# Talk2Mind — A Multimodal AI-Based Mental Well-Being Assessment and Support System

Talk2Mind is an intelligent virtual mental-wellness assistant that fuses **facial expression
analysis**, **speech emotion analysis**, and a **questionnaire-based assessment** to produce a
0–100 **Mental Well-Being Score**, a risk category, and **personalized recommendations**.

> ⚠️ **Disclaimer**: This is an educational/self-awareness project, not a diagnostic or clinical
> tool. It does not replace professional mental health care. The questionnaire is inspired by
> (but is not a validated substitute for) PHQ-9/GAD-7 screening instruments.

## Why a synthetic dataset?

Collecting real facial video, voice recordings, and mental-health questionnaire responses from
human subjects requires informed consent and ethics review, and is outside the scope of what can
be generated automatically. `data/generate_dataset.py` instead generates a **label-consistent
synthetic dataset**: a hidden per-user "distress level" drives correlated, plausible facial cues,
speech cues, and questionnaire answers — similar to how real distressed individuals show
correlated behavioral signals. This lets every downstream model (facial/speech classifiers,
fusion, scoring) be trained and evaluated end-to-end.

**To swap in real data later:** replace the CSVs in `data/` with real extracted-feature tables
that follow the same column schema, and every downstream script keeps working unchanged. The
`extract_features_from_video()` / `extract_features_from_audio()` stubs in the model files show
exactly where real OpenCV/librosa-based extraction would plug in.

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Generate the synthetic dataset
cd data
python generate_dataset.py --n_users 2000 --seed 42
cd ..

# 3. Train the individual models
python src/facial_emotion_model.py
python src/speech_emotion_model.py
python src/feature_fusion.py

# 4. Run evaluation
python src/evaluate_model.py

# 5. Launch the dashboard
streamlit run dashboard/app.py
```

## Model Performance (on synthetic data, see `data/evaluation_report.txt`)

- Facial Emotion Model: ~99% accuracy (synthetic features are strongly separable by design)
- Speech Emotion Model: ~75% accuracy
- Fusion MLP Well-Being Score Regression: R² ≈ 0.79, MAE ≈ 4.8 points (0–100 scale)
- Risk Category Classification (derived from predicted score): ~79% accuracy

These numbers reflect a synthetic, label-consistent dataset and should be re-validated once real
multimodal data is collected.

## Next Steps for a Production System

1. Replace synthetic facial/speech features with real extraction (OpenCV/MediaPipe + a
   fine-tuned CNN for faces; librosa/parselmouth + a fine-tuned BiLSTM/Transformer for speech).
2. Collect a small real, consented pilot dataset to validate/re-calibrate the scoring weights.
3. Add authentication, secure storage, and a database (e.g. PostgreSQL) for user history.
4. Add clinician-in-the-loop review for High Concern / crisis-flag cases.
5. Conduct a fairness/bias audit across age, gender, and accent groups.
